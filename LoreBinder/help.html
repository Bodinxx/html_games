Getting Started

  • Initial admin bootstrap account is ADMIN / admin; rotate this immediately in controlled environments.
  • Use + Doc and + Folder in the sidebar to build your project structure.
  • Double-click a document in the tree to open it in tabs.
  • Project changes autosave after 5 seconds of inactivity.

Editor Modes and Tabs

  • Visual Layout mode: direct editable document pane plus rendered preview.
  • Plain Text mode: raw markdown/code editing pane.
  • The mode tabs now expand the active editor to full width to avoid split-pane clipping.
  • Document tabs let you switch between open files and close them using the × button.

Markdown Features

  • Variables: [var:key="value"] and use with [var:key].
  • Wrapper blocks: ::: wrap-left ... ::: for class-based layout wrappers.
  • Curly blocks: {{monster-stat-block,border-color:crimson ... }}.
  • Links: standard markdown links, local anchors, and doc:<documentId>#anchor.

Assets

  • Upload image assets in the Assets panel.
  • Use Copy Link to copy markdown image syntax into your clipboard.
  • Rename/Delete asset actions are available per file.

User Accounts and Roles

  • Account requests require User Name, Real Name, Email, Password, and requested role.
  • User status can be active or banned.
  • Roles include admin, primary_author, sub_author, and reviewer.
  • All user records are stored server-side in an encrypted JSON data file.

Admin Console

  • Open with the Admin button (admins only).
  • Review and approve/reject account requests.
  • Ban/unban users, delete user accounts, and reset project content/assets.
  • Last login date/time is tracked per user record.
